# Shijiucode Inspired Visual Redesign

> 日期：2026-07-04
> 状态：设计已批准，待实施
> 方案：B — 纸面搜索工作台
> 范围：首页视觉系统、核心导航外壳、搜索与筛选面板、链接卡片、移动底栏

---

## 一、背景

nav-site 当前首页仍保留上一轮 Aupale-inspired 深色视觉：大背景图、玻璃面板、绿色强调和电影感标题。这个方向已经解决了“有记忆点”的问题，但与产品文档中“友好、明亮、高效”的定位出现偏离。

用户指定参考站点 `https://shijiucode.cn/`，希望新的视觉方向更简洁、更有艺术感、低饱和。参考站点的可借鉴之处不是具体内容或图片，而是整体节奏：

- 暖米白背景，页面像纸面而非仪表盘。
- 灰蓝文字和淡蓝强调，色彩低饱和。
- 中文衬线标题建立气质，正文仍保持清晰易扫。
- 大留白、细边框、弱阴影，层级克制。
- 首页先给品牌短句、统计和精选内容，再进入列表。

本项目仍是工具导航站，核心任务是“5 秒内找到工具”。因此 redesign 不能变成个人博客，也不能降低搜索、筛选、卡片浏览效率。

## 二、目标

1. 将首页从深色电影感调整为浅色纸面感，贴近“简洁、艺术感、低饱和”。
2. 保留并强化搜索优先的导航站效率：搜索框、分类、筛选、结果卡片仍是主流程。
3. 用衬线标题和留白建立气质，用低饱和蓝灰完成状态表达。
4. 移除深色玻璃、绿色高亮和大图背景依赖，降低视觉噪音。
5. 确保移动端仍紧凑可用，避免变成只适合桌面展示的画册页。

## 三、非目标

- 不复制参考站的图片、文案、站名、版式细节或资源。
- 不重构搜索、排序、评价、收藏、认证、数据库和 API。
- 不引入新的设计依赖、字体包、动画库或图片服务。
- 不做大规模信息架构改版，不调整分类体系。
- 不把首页做成营销 landing page。

## 四、设计原则

### 4.1 纸面优先

背景使用暖米白与淡灰蓝，而不是纯白或深黑。页面应有轻微纸面感，但不使用重纹理、大渐变或装饰光斑。

建议色彩：

| 用途 | 建议值 | 说明 |
| --- | --- | --- |
| 页面背景 | `#f8f6f2` | 参考站暖纸色，避免冷白 |
| 卡片背景 | `#fffdfa` | 比背景稍亮 |
| 主文字 | `#3d4a5a` | 灰蓝黑，低对比但需可读 |
| 次文字 | `#6f7f91` | 保持 WCAG 可读性 |
| 边框 | `#d8e0e7` | 淡蓝灰 |
| 主强调 | `#5f84b2` | 低饱和蓝 |
| 强调浅底 | `rgba(95, 132, 178, 0.10)` | 标签、激活态 |

### 4.2 字体层级

标题可使用系统中文衬线栈，建立一点书卷气：

```css
font-family: "Noto Serif SC", STSong, "Songti SC", Georgia, serif;
```

正文和工具信息继续使用 Geist / system sans。导航站是工具型界面，不让衬线侵入卡片正文和密集列表。

### 4.3 艺术感来自节奏

本次不靠 3D、强 motion、玻璃拟态、霓虹、渐变文字或装饰球体制造高级感。艺术感来自：

- 首屏留白和横向分隔线。
- 标题与搜索区的纵向节奏。
- 统计信息的轻量排布。
- 卡片边界和文字层级的克制。

### 4.4 效率不退让

搜索框仍在首屏核心位置。分类、热门筛选、结果卡片必须在首屏或首屏下沿可见。卡片可以更轻，但标题、域名、精选、收藏、预览按钮不能变弱到难扫。

## 五、组件设计

### 5.1 `HomeHero`

当前状态：

- `nav-hero-bg` 使用深色背景图。
- 主容器 `min-h-[76svh] md:min-h-[78svh]`，视觉很戏剧化。
- 标题是大号三行文案，偏品牌表达。
- 右侧 `nav-glass` 指标面板偏玻璃拟态。

目标状态：

- 改为浅色 hero，背景使用暖纸色到更浅纸色的极淡纵向过渡。
- Hero 高度压缩到约 `46svh-56svh`，让首批结果更早出现。
- 标题改为更安静的中文衬线表达，例如“导航图谱”或“寻得合用之器”一类短句。
- 搜索框置于标题下方，成为最强视觉锚点。
- 统计信息从玻璃卡变成横向低对比文本组，类似参考站的文章/照片/天数统计节奏。
- 热门分类保持可点，但样式改为淡蓝灰纸面 pill。

验收：

- 桌面首屏能看到 hero、搜索框、统计和结果区开头。
- 移动端不出现超大标题挤压搜索框。
- 不再引用 `/visuals/nav-atlas-aurora.png` 作为 hero 背景。

### 5.2 `Header`

当前状态：

- 固定深色玻璃顶栏。
- 白色文字、绿色品牌 icon。

目标状态：

- 固定浅色半透明纸面顶栏，使用 `bg-[#f8f6f2]/86`、淡边框、弱 blur。
- 品牌文字用衬线或小型大写风格，颜色为低饱和蓝。
- 桌面导航文字为灰蓝，hover 使用浅蓝底。
- 主题切换保持可用，但按钮风格与浅色体系一致。

验收：

- 顶栏在浅色 hero 和结果区上都清晰。
- 移动端菜单按钮不与背景混成一片。

### 5.3 `SearchBar`

当前状态：

- hero 版本是深色透明圆角搜索框。
- 语义搜索激活态使用绿色。

目标状态：

- hero 版本改为纸面输入框：白纸底、淡蓝灰边框、轻阴影。
- focus ring 使用低饱和蓝，不使用绿色。
- 语义搜索按钮使用同一蓝色体系。
- 保持 `Cmd/Ctrl + K` 提示和 loading 状态。

验收：

- placeholder 在浅色背景上可读。
- focus-visible 清晰，键盘用户可定位。
- 语义搜索开关状态不只靠颜色，仍有 `aria-pressed`。

### 5.4 `AtlasPill` 与 `SearchExperiencePanel`

当前状态：

- 深色玻璃面板。
- pill 使用白色透明和绿色激活态。

目标状态：

- 搜索体验面板改为轻纸面容器，可保留小圆角，但不做玻璃。
- pill 默认：淡边框、透明或浅纸底、灰蓝文字。
- pill 激活：淡蓝底、蓝灰边框、主蓝文字。
- “清除”按钮保持弱化但可发现。

验收：

- 选中状态在低饱和体系下仍一眼可见。
- 面板不会像大卡片嵌套大卡片，保持工具栏感。

### 5.5 `LinkCard` 与 `ResultGrid`

当前状态：

- 卡片是深色透明 surface，spotlight hover。
- 绿色精选徽章、白色文字。

目标状态：

- 卡片使用 `#fffdfa` 纸面底、淡蓝灰边框、极轻阴影。
- hover 只做边框、标题色和轻微上移，避免 spotlight 光效。
- favicon 背景改为浅蓝灰小方块。
- 精选徽章使用低饱和蓝或灰蓝，不使用绿色。
- 描述/域名/更新时间保持扫读层级。

验收：

- 5 列桌面网格仍清楚，不因阴影或边框显得拥挤。
- 移动端单列卡片按钮不挤压标题。
- 收藏和预览按钮有 hover/focus 状态。

### 5.6 `MobileNav`

当前状态：

- 深色底栏，绿色顶部指示。

目标状态：

- 浅色悬浮底栏，淡边框、弱阴影。
- 激活项使用浅蓝底和蓝灰文字。
- 保持横向滚动，避免一次性重构为抽屉。

验收：

- 小屏底栏文字不溢出。
- safe-area 仍生效。

## 六、全局样式调整

主要修改 `app/globals.css`：

- 更新 `:root` 和 `.dark` 以外的默认视觉基调，首页优先浅色。
- 将 `html` 背景从深色改为暖纸色。
- 替换 `.nav-hero-bg`、`.nav-glass`、`.atlas-section-label`、`.nav-empty-state`。
- 将 `.card-hover` 的 hover 色从高饱和蓝/绿色混用，统一到低饱和蓝。
- 保留 `prefers-reduced-motion` 和高对比度规则。

暗色模式本轮只做不破坏处理，不重新设计暗色主题。若用户后续要求，可单独做暗色适配。

## 八、风险与回滚

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 浅色重构触及多个组件，局部深色 class 残留 | 中 | 先改全局 primitive，再逐个组件验证 |
| 对比度过低导致工具信息难读 | 高 | 主正文和卡片文字不低于当前产品文档要求 |
| 参考站博客气质过强，削弱导航效率 | 中 | 搜索框和结果区优先，hero 压缩 |
| 暗色模式出现不协调 | 中 | 本轮不做暗色主视觉，但保证无白底白字等破坏 |
| 7897 端口被代理或其他进程占用 | 低 | 优先尝试 `[::1]:7897`，冲突时记录实际可用地址 |

回滚策略：

- 设计文档可单独 revert。
- 实现阶段按组件提交，若视觉不满意可回滚对应提交。
- 不涉及数据库、依赖安装或生产配置，回滚成本低。
